# Data-Quality Audit — Wave 1 (10 groups) — 2026-06-15

10 read-only auditors, one per feature group, against today's live stream store. Findings are heavily
duplicative; systemic root causes are tracked in `docs/AUTONOMOUS_BACKLOG.md` (P1.0). Deleting/recollecting
today's data is authorized.

## Systemic findings (cross-group)

### 🔴 CRITICAL-1 — `points()` lag features 100% NaN on live (parity break)
`incremental.py` evaluates `points()` exprs on a single-minute frame → `close.shift(k>0)` is null.
Dead features: `ret_accel_{5,10,15,30,60}m` (return_dynamics), `consistent_direction_*` (momentum_consistency),
`efficiency_ratio_*`/`directional_efficiency_*` (efficiency) ≈ 29 features, all 100% NaN every minute.
Backfill `compute()` produces real values ⇒ **live ≠ backfill**. Fix: resolve point lags through the
lag/slice machinery with a slice deep enough for the max point lag (120m); add a points-with-shift parity test.

### 🔴 CRITICAL-2 — restart wipes live buffers → long-window collapse + post-warmup NaN
At 15:00 and 15:11 UTC (= the universe/latency deploy restarts), 100% of symbols had every reduction
feature NULL for one minute, then long windows (60m/120m) collapsed to equal each other and never recovered
(`realized_vol_60m==120m` for 100% of post-15:00 rows; `up_ratio_60m==90m==120m==180m` for 3910/3910 symbols
by 16:06). Root cause: incremental ring re-seeds from a shallow/empty buffer on restart; no warm-start from
the store. `nan_policy="warmup"` masks it. Fix: rehydrate ring from the last ~120m of the store on startup
(also needed for nightly relaunch). Then delete & recompute today's long-window data from backfill.

### 🟡 MED — duplicate (symbol,minute) rows (every group)
SPY/QQQ/IWM + ETFs written by all 8 shards → up to 8× byte-identical rows (~2% of rows). Benign if consumers
dedup; fix at write/compaction. Until then readers must `.unique(subset=["symbol","minute"])`.

### 🟡 MED — std=0 divide degeneracies — ✅ FIXED (e0c9957)
`volume_zscore_*` 0/0→NaN (constant-volume window, ~0.25%/window, 1047 symbols) and `bb_position_20m`
±inf (flat 20m window, 636 rows). Both now guarded (zscore→0.0, bb_position→null); warmup nulls preserved.
